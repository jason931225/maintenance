# mnt-platform-auth

T0.5 authentication platform crate.

## Scope

- Server-side passkey ceremonies use `webauthn-rs` 0.5.5.
- Registration/authentication ceremony state is stored in Postgres
  `auth_webauthn_ceremonies`; it is never held only in memory.
- Passkeys are stored as serialized `webauthn-rs` `Passkey` values in
  `auth_webauthn_credentials`, with credential IDs unique across accounts.
- Access tokens are JWTs signed with ES256 through `jsonwebtoken` 10.4.0 and
  its explicit `rust_crypto` backend feature.
- Refresh tokens are opaque values. Only SHA-256 hashes are stored in
  `auth_refresh_tokens`.
- Refresh-token rotation marks each token as used and creates a replacement in
  the same family. Reusing a used or revoked token revokes the whole family.

## Static RP-domain files

The app composition root should serve static JSON generated by this crate at:

- `/.well-known/apple-app-site-association`
- `/.well-known/assetlinks.json`

The JSON contains the iOS `webcredentials` app IDs and Android
`delegate_permission/common.get_login_creds` entries for passkey association.
Actual production team IDs, package names, and debug/release/Play signing
fingerprints are ops configuration and are intentionally not hard-coded here.
