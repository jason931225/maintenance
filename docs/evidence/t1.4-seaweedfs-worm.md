SeaweedFS caveat:

SeaweedFS 4.32 implements S3 Object Lock at object-version level. Deleting the
current key after versioning is enabled creates a delete marker, but deleting
the locked version by `versionId` returns 403 before `retain-until`, and a
same-key write creates a new version without mutating the locked version.

Independently re-verified by lead 2026-06-12 against a fresh container: 1 passed, 1.10s.
