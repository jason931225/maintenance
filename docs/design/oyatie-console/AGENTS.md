## 3. 핵심 시스템(재사용할 문법)
- **핀/창 모델(카드·패널 공통)**: 헤더 드래그=팝아웃(자유 플로트) · 더블클릭=핀(분할 패널, 본문 `padding-right`로 실제 공간 확보) · 트레이 드래그/버튼=최소화. hover 3버튼 컨트롤=핀·최소화·X. behavior는 `cardVal`/`cardToolVals`/`cardGrab`/`cardPinRight`/`cardRestoreDefault`(1곳), 상세 패널 chrome은 단일 `<sc-for panels>` 헤더(1곳). 참조 핀=`cardFloat`(fixed+visible, 화면 이동 유지). 쿼드런트 핀=`panels`+`snapTo`.
- **목록**: J/K/Enter, 다속성 검색, 열폭 드래그, 공유 트랙 정렬, 끝 여백+페이드.
- **개인 수신함/passkey**: `inboxDocs`(state) · `pkModal`/`pkPhase` · `pkStart→pkAuth→inboxConfirm`(열람=수령 증빙 기록). 법적 문서 `legal && !confirmed` = 잠김.
- **연차촉진/노무수령거부**: `lvPromotePush`(1·2차 AP-)·`lvRefusalPush` → `apprMySubs`(상신함) + 대상자 개인 수신함 전달(수령확인 대기).
- **감사 백본**: `logEvent(partial)` — 모든 상태 전이가 호출(who/what/when/where/how/on-what/decision/integrity + 데이터분류 + `deviceCtx` + seq 해시체인). `screen:"audit"` 라이브 피드. 상태 전이는 반드시 이벤트/역참조 남김. 토스트에 결과+경로 명시.
- **토큰 문법(앱 전역)**: `tokenParse/tokenRender`(표시) + `tokenType/tokenKey/tokenPick`(컴포저 자동완성) + `tokenCatalog/tokenVisible`(PBAC 후보 게이트 — covert·등급초과 제외). @멘션·#개체·!코드·바코드·날짜. 새 입력창에 이 컴포저를 붙여 확장.
- **근태 타임라인**: `buildAttDayTL`(오늘 = 계획/실적 2트랙 시간축, 상태=개체). **월간(`buildAttMonth`)도 계획 vs 실적**: 상태 어휘 `lv`(유급휴가)·`pa`(승인 부재=무급/신청 승인, 계획 부재·차분)·`ab`(무단결근=실적 이탈)·`abc`(결근·대근 대체=결원 커버). `pa`/`lv`는 출근율·결근 집계 제외(승인), `abc`는 본인 결근 계상 유지. `abc`+대근 배지(person `subBadge`·노드 `coveredIn`)는 `s.subAssignments`(coveredWho) 라이브 반영 — renderVals `mData`에서 계산(캐시 무효화 불요). **대근/인력풀**: `WORKFORCE_POOL`·`subOpen/subAssign`(결원→인력풀 배정→AP-+타임라인 fill-in). **직원-일자 개체**: `openEmpDay`(계획·실적+그날 감사+급여영향+커뮤니케이션, 분류별 게이트). **기록물 등재**: `docRegOpen/docRegSubmit`(IN- 등재대기→승인).
- **자동화**: `workflows`/`schedules` + `wfRun/wfSimulate/schRun`(실행 시 logEvent·실제 AP- 생성). `screen:"auto"`.
