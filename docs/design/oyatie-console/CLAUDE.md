## 핵심 원칙 (요약 — 상세는 DESIGN.md)
- **모든 것이 개체(Object)다**: 사람·조직·문서·계약뿐 아니라 **액션·이벤트·현상**(결재·열람·무단결근·결혼·계약체결)도 개체. 온톨로지 3계층 = 의미(정체성)·동작(이벤트/전이)·역학(정책·자동화·파생분석)이 한 개체에 겹쳐 있다.
- **개체 체인**: 계약(C-) → 포지션(정책 프리셋) → 공고 → 지원자 → 직원 → 타임테이블/근태 → 연장근로 → 급여 → 분석 → 계약 수익성 환류. 새 기능은 이 체인의 상·하류 연결을 반드시 제공.
- **개체 생애주기 표준 (Draft→Archive)**: 대부분의 업무 개체는 정책형 파이프라인 — **초안→상신·검토→승인·게시(민감=passkey)→활성→개정(버전)→보관→폐기** — 를 따른다. 모든 전이=감사·버전+롤백·PBAC·상태 칩/스텝퍼. **거버넌스**(§3.9.1): 미래 발효일(effective-dating)·영향분석 사전점검·maker-checker(SoD)·전결규정(DoA)·참조무결성 정산 게이트·변경 동결창·**보관=숨김(하드삭제 금지)**. 파괴적 작업(폐지)은 의존 개체·법정 정산 완료 후에만. 신규 기능은 단계를 명시, 기존은 점진 retrofit. (DESIGN §3.9)
- **내부통제·가드레일 (예방 우선)**: 무권한·미검토·오배포 방지. 모든 액션은 ① 권한·자격 게이트(least privilege·미권한 미노출) ② 셀프 체크리스트 ③ 피어 검토(four-eyes·자기검토 차단) ④ 결재(SoD) ⑤ **배포/egress 게이트**(승인/게시 + 권한일 때만 외부 발송·내보내기) ⑥ 탐지(감사·알림)를 통과. fail-closed·자기승인 금지. (DESIGN §3.10)
- **콘솔 전역 Cedar PBAC**: 화면·카드·행·액션·검색·집계 전부 정책 평가 결과. "전체"=인가 법인 합집합. 권한 있어도 열람은 기록. 비밀 구역은 deny-by-omission.
- **자동화=결정적 또는 수동(§4-28 · 무AI)**: 배정·인수인계·위임·판정은 결정적 규칙(같은 입력=같은 출력·감사에 규칙 명시) 아니면 수동(조율 큐·four-eyes). AI/LLM 판정 금지.
- **no-code 우선 · 비주얼 · 개체 기반**: 정책·워크플로·자동화·예약작업 전부 자연어/블록 캔버스 + 시뮬레이션·되돌리기.
- **체크리스트 게이트(§4-29)**: 중대 액션=프리플라이트(자동 판정은 자동 체크·사람 판단만 attest·fail-closed)+포스트플라이트(실행 검증 자동 기록·이상=알림·되돌리기).
- **입력 불변식(§4-27)**: 폼은 사용자를 신뢰하지 않는다 — 정형 필드=입력 정규화 마스크(잘못된 형식이 존재 불가), 필수=fail-closed 게이트, enum=N+1 직접 입력, 개체·인물 선택=타입어헤드(열거 금지·프로덕션 규모 기준), placeholder=일반 키보드 문자·실무형 문구, 생성 폼=하류 효과 1줄 고지.
- **깊이 우선(deep not wide), 예외 우선, 설명용 UI 금지**: 상태=칩, 기능 없는 텍스트 금지. **부제·설명 캡션 금지** — 제목 밑 서브타이틀, 기술스택/프로토콜 캡션(예: "mox · IMAP4/SMTP"), "이 열람은 감사됨" 류 메타 안내, 칩·아이콘이 이미 보여주는 것을 반복하는 산문은 모두 제거(경고·에러 등 실제 액션을 유발하는 문구만 허용). 상세 보기 기본은 핀 패널.
- **전 직원 시스템 + 모바일**: 관리자 전용이 아님. 셀프서비스(본인 급여·개인 inbox)·passkey 수령확인. 커뮤니케이션은 rail↔main 승격.

## 안티패턴 — 절대 금지 (반복 위반 주의)
1. **설명 캡션·서브텍스트 금지**: 제목 밑 부제, 기술스택/프로토콜 캡션(예: "mox · IMAP4/SMTP"), "이 열람은 감사됨" 류 메타 안내, 칩·아이콘이 이미 보여주는 것을 반복하는 산문 — 전부 삭제. 상태는 **칩**으로, **실제 액션을 유발하는 문구(경고·에러)만** 허용.
2. **기능 없는 텍스트·데이터 슬롭 금지**: 클릭/드래그/필터 기능이 없는 라벨·배지·숫자·통계·아이콘을 넣지 않는다. 명사는 개체(클릭/드래그)이거나 아예 없다.
3. **AI 슬롭 비주얼 금지**: 그라디언트 배경, 이모지(브랜드 제외), 둥근 컨테이너+좌측 컬러 보더 액센트, 남용 폰트(Inter/Roboto/Arial). 색은 토큰·브랜드에서만.
4. **큰 숫자 KPI 카드 금지** → 컴팩트 1행 스탯 바(DESIGN §4-11).
5. **채우기 콘텐츠 금지**: 여백을 채우려 더미 섹션·설명을 추가하지 않는다. 부족하면 레이아웃으로 풀거나 사용자에게 묻는다.
> 원칙: **UI는 스스로 설명되어야 한다. 설명이 필요하면 나쁜 UI다.** (DESIGN §4-12)
