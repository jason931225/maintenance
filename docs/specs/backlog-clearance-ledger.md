## Generated-client and contract rules

1. API schema/contract changes must be made at the canonical server contract source, then regenerated with the repository scripts (`scripts/generate-kotlin-client`, `scripts/generate-swift-client`, and existing contract roundtrip checks as applicable).
2. Do not manually edit generated Swift/Kotlin clients or generated OpenAPI artifacts to make tests pass.
3. Cross-lane API changes require an owner: the lane changing the server contract owns regenerated clients, contract tests, and mobile/web consumer updates.
4. If generated output is stale, the lane must record the exact regeneration command and diff evidence; if generation fails, checkpoint a blocker instead of hand-patching generated files.
5. Data imports must preserve raw-file lineage outside normalized tables and export only standardized, typed schemas.
