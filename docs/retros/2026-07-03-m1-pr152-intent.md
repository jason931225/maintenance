## What worked
- **Verify-before-accept caught real defects**: the adversarial verify stage found a live-red e2e
  (spec unrunnable in the prod-build harness), a plan-required-but-deferred release probe, and 3
  process-handling bugs — all before merge. The fix loop closed them in one round.
- **Fail-closed gates proved themselves live**: `ci-gate` blocked deploys for CI-cancelled SHAs
  during rapid merges, and the brand-new `release-probe` blocked `bump-digests` on its own first
  (defective) run. Both "failures" were the safety design working.
- **Exercising the dev server for the first time surfaced dormant bugs**: a React StrictMode
  boot-refresh race that wedged all `vite dev` sign-ins, and bacon crashing without a TTY. Mock-mode
  had been hiding real-path defects — the strongest argument for real-over-mock.
- **Outcome proof**: within hours of the real stack landing, the same contributor whose mock-data PR
  started this (#152) submitted a surgical, CI-green 2-file fix (#161).

## What to fix in the process
- **One live instance per worktree.** A background CI-watcher continuation plus a later fix-task
  assignment to the same agent created two concurrent writers on one checkout (silent edit
  reversions). The worker's response — stop, never force-push, escalate with evidence — is the
  correct guardrail and should be codified in AGENTS.md (M3).
- **Plan-required items are not deferrable by executor judgment.** The release-image probe was
  flagged "follow-up" and accepted in the moment; the verifier caught that the plan text said
  *required*. Leads must check deferrals against plan text, not plausibility.
- **Workflow-file changes can't prove themselves**: `image-release.yml` is not in its own `paths`
  filter, so #162's fix needed a manual `workflow_dispatch` to verify. Consider adding
  `.github/workflows/image-release.yml` to the filter.
- **Stale-context grounding is the recurring failure mode**: a stale local `main`, a stale spec
  Technical Context (missed the existing `ops/` stack and ADR-0018 runtime spine), and a stale
  deploy-gating memory all had to be corrected mid-flight by adversarial review. Ground plans
  against `origin/main` at write time, and re-verify memories before load-bearing use.
