# Best-in-Class Benchmark Matrix — Executive Summary

**14 modules × 7 primary vendors (+ secondary references), read through 5 independent lenses, adversarially reviewed (4 verifier passes, 23 web checks + code adjudication).**

Each `<module>.md` in this directory is a self-contained benchmark: an evidence-based read of our console (`web/src/console/**` + backend crates + `docs/program/console-program-ledger.md`), a capability matrix vs Palantir Foundry · Slack · Microsoft Teams · Asana · n8n · Rippling · SAP (with ServiceNow/Workday/Zendesk/Vanta/Drata/Windmill/Temporal/Cedar/Purview/Korean-groupware as module-specific references), per-vendor "how they'd build our module" paragraphs, a **Cross-cutting lens findings** section (the 5 lenses woven in + adjudicated contradictions), and a ranked steal-list. The 5 lens documents are in [`lenses/`](./lenses/).

Every vendor claim is labeled `[V]` verified (source URL) or `[I]` inferred; every "Ours" claim is code-grounded. The adversarial review found a **very low fabrication rate** — the corrections applied were narrow (a handful of stale figures, four wrong migration numbers in the lenses, one refuted Teams-Shifts claim, a layer-mislabel in `field`, a Korean-statute citation, and two convergent steal-list omissions).

## Per-vendor patterns — what each vendor is systematically best at

- **Palantir Foundry — the ontology reference; our explicit north star.** Object/property (cell-level) policy; **Actions as the sanctioned mutation verb**; branching/proposal-as-PR schema governance; **markings that propagate down lineage**; search-around exploration + saveable Layouts; Functions-on-objects; Automate condition→effect where automation and humans fire the same verb. Nearly every Foundry "steal" is *"finish what we half-built."* Weakness for us: no 전자결재/근로기준법 model, US-gov origin.
- **SAP (S/4HANA · SuccessFactors · GRC · FSM · SBPA) — enterprise rigor, at the cost of weight.** The **document principle** (immutable balanced GL, ACDOCA), OB52 period control; the **SoD ruleset + risk-at-grant + UAR recertification** benchmark (GRC); release-strategy **amount/attribute-driven routing** + flexible-workflow parallel approval; effective-dated master data + **first-class Positions/org units**; AI dispatch scheduler + **Crowd Service** (FSM); decision-table approver determination (SBPA). Steal the *invariant discipline*, reject the config-priesthood.
- **Rippling — the attribute-cascade HRIS.** The **employee graph**; **attribute-bound permissions that auto-update on transfer** (Supergroups); **Workflow Automator** (any field is a trigger); rule-based PTO accrual; geofenced clock-in bound to payroll; 90-second pay-run. Best at lifecycle-linked automation. Weakness: US-payroll-centric, Korea via EOR overlay (no 근로기준법/전자결재 native).
- **Slack — collaboration + triage velocity.** Activity-view triage (filter pills, clear-all); **swarm threads** (26% faster close, 64% backlog cut); **one-click inline decide**; Workflow Builder conditional branching (≤15 branches, ≤10 rules each, Business+/Grid); link unfurl; Discovery API + Legal Holds + DLP (Enterprise Grid). Best at the human-coordination + notification-terminal loop; shallow on structured objects and governed config.
- **Microsoft Teams — the frontline + enterprise-compliance hub.** Activity feed; native **Approvals app + e-signature**; **Shifts** (geofenced clock-in + handover; note: approval marks unavailable, does *not* auto-remove); Purview retention/eDiscovery; 1,000+ Power Automate connectors. Best at in-flow approvals + frontline mobile; data scatters across M365, 전자결재 needs third-party add-ons.
- **Asana — no-code work-graph ergonomics.** Customizable Home + My Tasks + Inbox; Forms→task intake; **Rules + Bundles** (reusable automation pushed to many scopes); reference custom fields; immutable audit log. Best at time-to-first-chart + citizen-config; capped (100 fields/project), no as-of, no governance depth, explicitly *no* custom retention / legal hold.
- **n8n — the open automation reference.** Free-form node DAG; 400+ connectors + HTTP + code node; **inbound webhook trigger**; Error-Trigger workflow + per-node execution debugging; data pinning. Best raw engine + debugging; untyped JSON items, no governance, prunes-not-preserves (the antithesis of custody).
- **Secondary references the lenses surfaced:** **ServiceNow** (configurable agent workspaces, GRC audit workspace, FSM Dispatcher Workspace) · **Workday** (BP-framework, effective-dating, Related Actions) · **Vanta/Drata** (continuous control-monitoring, hourly/daily) · **Windmill** (`permissioned_as`) · **Temporal** (durable event-sourced replay) · **Cedar** (policy templates, schema validation) · **Purview** (retention/disposition/eDiscovery) · **Zendesk** (macros, custom objects) · **Zapier** (linear simple-case path, 8,000+ apps) · **Concur/NetSuite** (1-click / bulk approve) · **Korean groupware 더존·다우오피스·한비로** (결재선, 함 IA, 전결/대결/후결).

**The one structural fact across all vendors:** every commodity vendor's extensibility is **outward** (connectors, webhooks, apps, code) and **ungoverned**; ours is **inward** (extend the ontology model) and **governed**. The four things a power user loses moving to us — inbound webhook, connector catalog, code node, app platform — are exactly the four we deliberately didn't build. The thing they gain — every automation runs the same Cedar-gated, four-eyes-published, as-of-audited Action a human runs — is the moat.

## Where WE are ahead (evidence-based)

1. **Governance is the substrate, not a bolt-on.** FORCE-RLS tenant floor tested as the real `mnt_rt` role (not a BYPASSRLS superuser); Cedar deny-by-omission down to the *field* (property-policy → partial-eval → residual → SQL `WHERE`, composed with the RLS floor, fail-closed on any untranslatable term); four-eyes `CHECK(approver_id <> requested_by)` at the DB. Most vendors bolt governance ON; ours is the spine every surface renders through.
2. **Tamper-evident hash-chain audit (L20) under every mutation** — crate-owned canonicalization + gap-proof CoverageGap verify + corrupt-seal→verdict, not just an append log. No vendor ships a customer-facing per-revision hash-chain by default; it is a stronger integrity guarantee than any vendor evidence store.
3. **Config-as-governed-object.** Dashboards, SLO settings, console views (and, on the roadmap, policies + approval templates) are themselves versioned ontology instances with draft→approve→effective + rollback + as-of + four-eyes. Foundry config isn't a first-class versioned business object; no support vendor governs its SLA/SLO config that way (a config change here is itself four-eyes-able).
4. **Object-level as-of / bitemporal on instances.** `get_as_of` + `history` + per-revision `hashVerified` reconstruct the *whole object* as-of T. Foundry's temporal story centers on time-series properties + edit-history transactions; a uniform object-level reconstruction is our native primitive.
5. **Automation fires the same governed Actions as humans.** No shadow service-account privilege — every automation edit inherits Cedar + four-eyes + audit + fixity. n8n/Zapier run under service accounts; ours is indistinguishable from a governed human edit.
6. **Honest-by-default UI (a design law, not a preference).** Unbacked sections are *omitted, never placeholdered*; axis-truncation is code-governed and forces a mandatory "축 절단" warn chip (anti-deceptive-chart law — no vendor ships this); CVaR/tail-risk ships in a mainstream ops dashboard; the deliberate **NO-AI** stance keeps every surface deterministic and auditable.
7. **Korean-statutory depth is an uncatchable moat.** 전자결재 (결재선/전결/대결/후결/반려, post-finalization rejection + compensation), 연차촉진 §61 round FSM, the 노무수령거부/수령확인 notice-and-receipt flow, the 4대보험/NTS/severance payroll kernel (compile-error on omission of the ordinary-wage floor), per-shift 근로계약, PIPA/개인정보보호법, and Group→법인→branch→worksite scoping woven into every capability. Global vendors force-fit or omit all of it; SAP has the approval spine but a heavyweight, generic 전자결재.
8. **Real, RLS-tested finance period lock.** `period_lock.rs` (+ migration `0107`) blocks ledger writes into a closed period, tested as `mnt_rt` — production-grade, and a genuine edge over Rippling's "feed the connected GL" and Palantir's modelled guard. (The governance & data-model lenses under-credit this; it stands.)
9. **The most-collapsed leave surface in the field** — co-located personas (self-submit + team-decide + 촉진 + ledger) on one screen with inline SoD-by-omission — beats Workday's worklet spread. A pattern to *propagate*, not fix.

**Honest caveats that keep this list credible:** the engine **out-models the field but is under-populated** (~4 registered types; projected-action dispatch `NotWiredYet` — hence steal #1/#3). Several flagship strengths are **intent, not yet law** — most notably the dashboard/overview "drill-everything" is wired to react-router paths that dead-end in the `state.screen` console shell until rewired. And most of the Korean IA moats (the 결재함/상신함 함-IA, the dispatcher board, the interactive comms rail, the command palette) are correctly identified in nav but **unbuilt**. The steal-list is dominated by **populating and wiring** what the engine already supports, not by new architecture.
